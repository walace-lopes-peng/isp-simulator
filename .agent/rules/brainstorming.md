# Regras de Brainstorming & Design Backlog

Quando o usuário sugerir novas mecânicas, ideias de UI, ou mudanças no jogo que ainda não foram aprovadas para o Roadmap ou Blueprint Oficial, siga estritamente estas regras:

1. **Inserção no Icebox**: Novas ideias sugeridas em chat devem ser inseridas obrigatoriamente no arquivo `BRAINSTORM.md` na raiz do projeto.
2. **Status Padrão**: Toda nova ideia deve ser categorizada sob o status "🟢 Livre" e associada ao seu pilar de gameplay correspondente.
3. **Proibição de Issues Autônomas**: É estritamente proibido graduar uma ideia livre para uma Issue do GitHub sem a aprovação explícita e em prompt do usuário. 
4. **Referência**: Avalie impactos baseados na métrica "Alta/Média/Baixa" usando a Matriz de Priorização descrita no final do arquivo `BRAINSTORM.md`.
